# Final Project Expectations

Brother Clements reviewed the four major components of the final project grade.

## 1. Working Product

Students are expected to deliver a functioning application.

- Every team member should contribute meaningful functionality.
- Delivering only one feature may result in a reduced grade.
- Students with significant additional contributions may discuss an individual grade claim if appropriate.

---

## 2. Documentation

Documentation should be completely updated before final submission.

Key points:

- Documentation saves significant development time.
- Future developers should be able to understand the project without reading every line of code.
- Documentation should accurately reflect the final implementation.

---

## 3. Individual Participation

Participation will primarily be evaluated through GitHub.

Evaluation includes:

- Commit history
- Merge history
- GitHub Insights
- Meeting attendance
- Product assessments

Brother Clements demonstrated how he reviews each student's GitHub contribution graph and repository activity.

Students should ensure:

- All completed work has been merged.
- Contributions are visible on GitHub.
- No completed work remains isolated in personal branches.

---

## 4. Final Presentation

Every team member is expected to participate.

The presentation should include:

- The problem being solved
- Customer interviews and requirements gathering
- Existing solutions and why they were insufficient
- The team's solution
- Customer validation
- Demonstration of the working product
- Features completed
- Stretch goals
- Lessons learned throughout development
