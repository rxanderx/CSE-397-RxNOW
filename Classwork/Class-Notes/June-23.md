# Importance of Frequent Merges

Brother Clements shared industry experiences where developers waited until the final day to merge their work.

This resulted in:

* Massive merge conflicts
* Failed regression tests
* Emergency weekend fixes

Recommended workflow:

1. Finish a task.
2. Test it.
3. Merge immediately.
4. Run regression testing.
5. Allow QA to begin testing as early as possible.

Frequent merging reduces project risk.

# Software Architecture Lecture

The second half of class connected concepts from across the Computer Science curriculum.

## Input → Process → Output (IPO)

Programming can be viewed as three primary stages:

* Input
* Processing
* Output

This model ties together nearly every programming concept students have learned.

---

## Functional Programming Concepts

Brother Clements explained that nearly every function can be categorized as one of three operations.

### Filter

Produces a subset of data.

Examples:

* Filtering odd numbers
* Removing records
* Selecting matching values

---

### Map

Transforms one value into another.

Examples:

* Reverse strings
* Convert values
* Modify collections
* Append new values

---

### Reduce

Produces a cumulative result.

Examples:

* Counting
* Summation
* Totals
* Aggregation
