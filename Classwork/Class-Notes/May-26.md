# Software Development Topics

## Software Development Methodologies

### Waterfall Model

A traditional sequential development process consisting of:

1. Requirements Gathering
2. Design
3. Implementation
4. Testing
5. Deployment
6. Maintenance

**Characteristics:**

* Linear progression
* Extensive planning upfront
* Difficult to accommodate changes late in development

---

### Spiral Model

An iterative model that combines development phases with risk analysis.

**Key Features:**

* Risk identification and mitigation
* Iterative development cycles
* Customer feedback at multiple stages
* Suitable for large, complex projects

---

### Agile Methodology

Agile emphasizes flexibility, collaboration, and customer satisfaction.

**Core Values:**

* Customer collaboration
* Responding to change
* Working software
* Continuous improvement

#### Key Agile Principles

1. Satisfy customers through continuous delivery.
2. Welcome changing requirements.
3. Deliver working software frequently.
4. Maintain a sustainable pace.
5. Build projects around motivated individuals.
6. Encourage self-organizing teams.
7. Regularly reflect and improve processes.

**Industry Note:**

Technology and development practices typically evolve every 5–10 years, making adaptability essential.

---

### V-Model

An extension of the Waterfall Model where testing activities correspond directly to development phases.

| Development Phase   | Testing Phase       |
| ------------------- | ------------------- |
| Requirements        | Acceptance Testing  |
| System Design       | System Testing      |
| Architecture Design | Integration Testing |
| Module Design       | Unit Testing        |

**Benefit:**

* Testing is planned early and integrated throughout development.

---

### DevOps

DevOps integrates software development and IT operations into a continuous cycle.

#### Typical DevOps Lifecycle

1. Planning
2. Development
3. Building
4. Testing
5. Release
6. Deployment
7. Operations
8. Monitoring

**Goals:**

* Faster delivery
* Increased reliability
* Continuous feedback
* Improved collaboration

---

## Story Estimation

### Fibonacci Estimation

Agile teams estimate complexity using Fibonacci numbers:

```text
0, 1, 2, 3, 5, 8, 13, 21, ...
```

### Important Concepts

* Estimates represent **complexity**, not hours.
* Teams use the **Delphi Method** to reach consensus.
* Velocity is tracked across sprints.
* Historical velocity helps predict project timelines.

### Estimation Adjustment

To compensate for optimism bias:

> Multiply estimates by π (Pi)

This provides a more realistic prediction of effort and schedule.

# Career Readiness Topics

## Understanding the Job Market

### Key Statistics Discussed

* Approximately 80% of jobs are never publicly posted.
* Many published positions are already internally filled.
* Online applications have very low success rates compared to networking.

### Implications

Networking provides access to:

* Hidden opportunities
* Hiring managers
* Industry insights
* Professional referrals
