# 🔄 Integrated SDLC + DevOps + Roles + Deliverables

## 📊 DevOps / SDLC Alignment with Roles and Deliverables

| DevOps Phase | SDLC Phase                                 | Primary Roles                               | Key Activities                                            | Required Deliverables                                                      |
| ------------ | ------------------------------------------ | ------------------------------------------- | --------------------------------------------------------- | -------------------------------------------------------------------------- |
| **Plan**     | ConOps, Requirements, Architecture, Design | Product Owner, Scrum Master, Tech Leads, QA | Define scope, requirements, architecture, sprint planning | ConOps, SRS, SDD, Product Backlog, Sprint Plan, API Contracts, Data Model  |
| **Code**     | Implementation                             | Tech Leads, Scrum Master, QA                | Develop features, enforce MVC, implement APIs             | Source Code (FE/BE/DB), MVC Structure, API Implementations, Code Reviews   |
| **Build**    | Implementation                             | CM/DM, Tech Leads                           | Integrate code, compile in CI environment                 | CI Pipeline, Build Logs, Build Artifacts                                   |
| **Test**     | Verification & Validation                  | QA, Tech Leads, Product Owner               | Execute unit, integration, system, acceptance testing     | Test Plan, Test Cases, Test Results, Defect Log                            |
| **Release**  | Transition                                 | CM/DM, QA, Product Owner                    | Package product, finalize documentation                   | Version Description Document (VDD), Release Notes, User Documentation      |
| **Deploy**   | Deployment                                 | CM/DM, Tech Leads, QA                       | Deploy to target environment, validate system             | Deployment Scripts, Deployed System, Smoke Test Results, Deployment Report |
| **Operate**  | Operations & Maintenance                   | Product Owner, Scrum Master, Tech Leads     | Collect feedback, support users, refine backlog           | User Feedback Log, Updated Backlog, Retrospective Notes                    |
| **Monitor**  | Operations & Maintenance                   | Tech Leads, QA, CM/DM, Product Owner        | Analyze logs, monitor performance, diagnose issues        | Logs, Metrics, Error Reports, Incident Reports                             |

# 🔁 Version-Based Delivery Alignment

## 🔹 Version 0.0 — Plan Phase Dominant

* **DevOps Focus:** Plan
* **SDLC Focus:** ConOps, Requirements

### Deliverables:

* Requirements Elicitation
* Software Requirements Specification (SRS)
* Initial VDD (0.0)

---

## 🔹 Version 1.0 — Plan → Code → Build

* **DevOps Focus:** Plan, Code, Build
* **SDLC Focus:** Architecture, Design, Implementation

### Deliverables:

* Software Design Document (SDD)
* API Contracts
* Initial Working Build
* VDD 1.0
* Product Package v1

---

## 🔹 Version 2.0 — Full DevOps Cycle Introduced

* Product Package v2

---

* Product Package v3

# 🎯 Key Teaching Alignment

* **Every version = one full DevOps loop (increasing maturity)**
* **Roles are active in every phase, but ownership shifts**
* **Deliverables evolve from documentation → working system → operational system**
* **Students experience both:**

  * Structured SDLC artifacts
  * Iterative DevOps execution

---

# 🧠 Simple Rule for Students

> “Each version must demonstrate a working system AND the evidence of how it was built, tested, deployed, and improved.”
