# Elicitation Checklist

Please complete the following checklist as a gitHub Issue and provide links to your project's repo


* [Brainstorming Analysis](./Brainstorming/Readme.md)
   - [ ] 20 different features
   - [ ] What is the critical MVP for the 1st or next version of software? (aka, what needs to be done this semester?)
   - [ ] What information do you need to know from the customer?
* [Observation and Document Analysis](./DocumentAnalysis/Readme.md)
    - [ ] Come up with 20 different requirements that the project should implement.
    - [ ] What are the requirements that the system must do?
    - [ ] What are the existing requirements, if implemented?
    - [ ] What is missing in the documentation?
    - [ ] What would like to know more about?
* [Introspection Analysis](./Introspection/Readme.md)
    - [ ] Ponder what are the wants and needs of the user, customer, and stakeholders.
    - [ ] What are the pain points of the customer?
    - [ ] What are the types of users that might use the product? How will they use it?
    - [ ] What other stakeholders are needed?
    - [ ] What are the knowns and unknowns? What gaps are missing?
    - [ ] What are the risks? 
* [Empathy Analysis](./Empathy/Readme.md)
    - [ ] Users: Document five different user types that would be using, maintaining or modifing the system. Examples include: admin, guest user, registered user, managers, accountants, etc
    - [ ] User Interfaces: Document ten different user interfaces (i.e. wireframes) that the user would interact with. (i.e. Account creation, User Login, feature pages, etc)
    - [ ] Workflows: Data in motion: Document ten different use cases or scenarios for the software product. Including, account creation, user logging in, and starting home page workflows.
    - [ ] Data Objects: Data in Motion: Documents different data models that would be used to store the user's actions.
    - [ ] Data Objects: Data at Rest: How will the user's actions effect the state of the system?
    - [ ] What are the different user's roles?
    - [ ] (Front-end) How are the user interface's connected?
    - [ ] (Back-end) What logical decisions need to be made within the workflows (i.e. control flow, security)?
    - [ ] (Database) How are the data models connected?

* [Project Overview Analysis](./OverviewAnalysis/Readme.md)
    - [ ] Create a model drawing of what you know about the solution?
* [Quality Characteristic Analysis and Model](./QualityModel/Readme.md)
    - [ ] Create three quality models
