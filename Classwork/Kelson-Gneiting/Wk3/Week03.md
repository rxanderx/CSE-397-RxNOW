# Week 3. SW Requirements Document(SRS)

## Objectives
- Define SRS scope and system boundaries for the semester deliverable.
- Elicit and document functional requirements for core features with clear use cases and acceptance criteria.
- Capture non-functional requirements (performance, security, availability, usability, maintainability, portability).
- Assign SRS sections to owners and set deadlines for first drafts.
- Establish a peer-review process and schedule for integrating sections into Version 0.0 — Requirements Foundation.
- Identify open questions and required evidence from the customer or stakeholders.
- Produce draft SRS sections ready for internal review by the end of the week.
