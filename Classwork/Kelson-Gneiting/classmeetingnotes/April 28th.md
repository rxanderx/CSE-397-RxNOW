## Meeting Minutes

* Date:
  Week 02

* Title:
  Week 02 - GitHub Workflow and Project Planning

* Purpose:
  To review GitHub version control practices, explain project planning requirements, introduce Agile task management concepts, and prepare teams for project development.

* The Agenda:

  * Items planned on discussing:

    * AWS and career readiness requirements
    * GitHub version control
    * Agile stories and task management
    * Team roles and planning
    * Project development workflow

  * Items actually discussed:

    * AWS certification accommodations for students already certified
    * Weekly status reports and project meeting requirements
    * GitHub branching, merging, and pull workflows
    * Use of GitHub Issues, Projects, Milestones, and Kanban boards
    * Agile stories, story points, and definitions of done
    * Team roles including Product Owner, Scrum Master, Tech Leads, QA, and CM/DM
    * DevOps lifecycle and software documentation requirements
    * Full-stack architecture, MVC, and CRUD concepts
    * Customer communication process and project repository setup

  * Jake

  * Nikki

* [Decisions](../Decisions/Readme.md):

  * Students with existing AWS certifications do not need to retake AWS training and may submit proof of completion.
  * Students should only merge into the assigned bi-weekly development branches.
  * No student work should be merged directly into the `main` branch.
  * All GitHub Issues must be associated with the project board and appropriate parent tasks.
  * Only the approved course milestones should be used within GitHub Projects.
  * Product Owners will serve as the primary point of contact with customers.

* Action Items:

  * Students should organize GitHub Issues under the appropriate parent tasks and milestones.
  * Teams should determine role assignments, including Product Owner.
  * Students should begin requirements elicitation once customer introductions are made.
  * Brother Clements will provide repository access and customer contact information.
  * Students should invite Brother Clements to team communication channels when appropriate.
  * Teams should continue completing weekly status reports, planning documents, and project tasking.
