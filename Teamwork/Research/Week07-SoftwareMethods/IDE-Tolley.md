## Introduction: 
* IDE is integrated development environment. It is the workbench with which development can be done. Without an IDE, development would essentially be impossible.

## Overview: 
* We will be comparing three different IDE's; Eclipse, Android Studio, and VS-Code.
## Strength and Weakness: 
* With Eclipse, it's great at debugging, finding build errors, and developing it's supported languages. The downside is that it is not as versatile as VSCode and it is still fairly heavy on resource use. It feels a little outdated compared to other IDE's. 
* VS-Code as mentioned is very versatile. If needed, one simply needs to download a particular extention to work with another language. Its something you can't go wrong with, but you could probably do better if there is a specialized IDE out there. It is great for one stop development, and its heavily supported by the community and developer itself. It can be lightweight by using only what you need. 
* Android Studio is a great choice for Android apps. It can be heavy, however, and use a lot of computer resources. If it is what you need, it works great and also has an emulator you can test your product on. It does start up and run a bit slower than other IDE's, so if its not what you need, it won't be your best option. 

# Useage and Applicability: 
* Eclipse works well with C, Java, or enterprise software. It can handle large projects particularly if they use Java framework.
* VSCode is so general purpose that it can be used in any situation, but not as well as a dedicated IDE in all likeliness. Great for multiple languages.
* Android Studio is geared towards Android development as the name implies. It is a heavier IDE and while it might handle other extensions, it is optimal for Android Apps, less optimal for anything else. 

# Comparison: 
* The best IDE is the one you are most comfortable with in my opinion. For most people that will look like a more generalized IDE such as VS. For those interested in making a mobile app, something like Android Studio would be an excellent choice. I don't see enough upside to Eclipse to recommend it in all honesty, perhaps I need to research more and find better strengths. Usage makes one more proficient and efficient, so whichever one you are comfortable with, I would stick with unless there truly is a feature only another one might provide.
  
# Summary: 
* For our project, at least while we are developing, android studio is the best option for me. As a team, we need to ensure compatibility across multiple devices so Android won't help there, but for me it's what I have and what I can use my phone with to test the actual app. I will need to research about deployment options for other devices to know better if another option is better. I used Visual Studio early on but when it came time to integrate and build, I thought Android Studio was superior in my situation.
